あなたはセキュリティ監査の専門家です。このプロジェクト（Riot.js SPA + Firebase + Cloudflare Workers）のコードベースを徹底的に調査し、脆弱性レポートを作成してください。

## 調査観点

### 1. XSS (クロスサイトスクリプティング)
- `innerHTML` の直接使用箇所
- DOMPurify (`sanitize.ts`) の使用漏れ
- ユーザー入力がサニタイズなしで DOM に挿入されている箇所
- Riot.js テンプレート内での `raw` 式の使用

### 2. 認証・認可
- Firebase Auth のチェックが欠落しているエンドポイント
- Cloud Functions で認証なしにアクセス可能な関数
- クライアント側のみの認可チェック（サーバー側で未検証）

### 3. 機密情報の露出
- ハードコードされた API キー、シークレット、トークン
- `.env` ファイルが `.gitignore` に含まれているか
- クライアントバンドルに含まれるべきでない情報
- Stripe のシークレットキーの取り扱い

### 4. Firestore セキュリティルール
- `firestore.rules` の許可範囲が広すぎないか
- 認証なしの読み書きが可能なコレクション
- フィールドレベルのバリデーション不足

### 5. 依存パッケージ
- 既知の脆弱性を持つパッケージ（`pnpm audit` の結果）
- 不要な依存パッケージ

### 6. ネットワーク・設定
- CORS の設定不備
- CSP (Content Security Policy) ヘッダーの欠如
- HTTPS の強制

### 7. インジェクション
- Firestore クエリへのユーザー入力の直接使用
- URL パラメータの未検証使用

## 出力フォーマット

各脆弱性について以下の形式でレポートしてください:

```
### [重要度: Critical/High/Medium/Low] タイトル
- **場所**: ファイルパス:行番号
- **問題**: 何が問題か
- **攻撃シナリオ**: どのように悪用されるか
- **推奨修正**: 具体的な修正方法
```

最後に重要度ごとの件数サマリーを付けてください。
